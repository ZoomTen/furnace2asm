Furnace2Asm
0.1

What is it, really?

Eh, something to convert Furnace project files into assembly for use in pokecrystal, pokegold, maybe even pokered (option to compile that maybe coming soon) if you so wish. Nothing special, really. I had a command line tool but I decided to upgrade it into a Windows (yes, Windows-only) GUI tool because of all the random-ass GBA binary hacking tools that use this mid 00s aesthetic, and I miss it a bunch. It was kino. It was a Python tool, and for it all I rewrote it in Nim. Go figure. :p

Major props, of course, go to Ward's wNim library for making this possible. I don't really wanna deal with VB6 just to make this silly app, lmao.

Why use this?

For the simple reason that Furnace actually emulates a Game Boy, so what you hear is a lot closer to what you get.

Any catches?

A few, actually:

  • I haven't yet looked into the new v158+ pattern format, and I don't wanna code in support for the old (< v127) instrument format (esp. for a quick project), so the acceptable versions are 127–157, which means only modules created between 0.6pre2 and 0.6pre4 work with this tool. Sorry. :(
  • Because it is expressly for converting Furnace project files to the GBC Pokémon disassemblies format, the project must only consist of a single Game Boy chip. Likewise, only Game Boy instruments are supported with this tool. Any wild FM instrument and it'll freak out...
  • If, in a pattern, the first note does not start at row 0, you should put a rest (OFF) note at row 0.
  • Looping is supported, but it's assumed that only one 0Bxx effect is present in the entire project. If the song does not loop, do not include a 0Bxx effect.
  • Duty cycles are only serviced through the 12xx effect.
  • Stereo panning is supported through the 08xy effect.
  • Setting the wave instrument is supported through the 10xx effect on channel 3.
  • The noise channel works like this: Notes are ignored, but instrument numbers are important. The tool does not currently support mapping these instrument numbers to drum set numbers from the get-go, but constants will be made for you so you can edit them in.
  • The song title is used for naming labels and constants. A song title like Cinnabar island will be transformed to Music_CinnabarIsland and CINNABAR_ISLAND.
  • Macros are not supported.
  • If you have a Bxx or Dxx effect on one of the channels, you should have it on the rest of them, otherwise channels would desync.
  • There may be some bugs I haven't yet accounted for.
